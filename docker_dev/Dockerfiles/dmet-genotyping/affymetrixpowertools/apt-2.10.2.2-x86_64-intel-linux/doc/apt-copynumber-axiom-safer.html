Introduction

apt-copynumber-axiom-safer is a program for detecting copy number variations (calculating copy number states) in specified genes.

Quick Start

The basic requirements for a run of apt-copynumber-axiom-safer are:

  • The .summary.a5 (or .summary.txt) and .report.txt files from apt-copynumber-axiom-safer. Note that for a large number of samples the program will run much faster using the summary.a5 file.
  • The .cn_models reference file.
  • The .cnbins.txt regions file.
  • The .controls.txt plate controls file.

Additional parameters may be specified on the command line or by means of a parameter XML file.

The example command below runs the program:

  apt-copynumber-axiom-safer \
      --analysis-files-path /library/file/path \
      --arg-file AxiomCN.xml \
      --out-dir out \
      --log-file ./apt-copynumber-axiom-safer.log

Alternatively, parameters can be set using command line parameters directly:

/nfs/swbuildtmp/devel/jbiela/Axiom/release_build/output/x86_64-intel-linux/bin/apt-copynumber-axiom-safer \
      --analysis-files-path /library/file/path \
      --reference-file PharmacoScan.20160620.cn_models \
      --summary-file /path/to/AxiomGT1.summary.a5 \
      --cn-regions-file /path/to/PharmacoScan.9.cnbins.txt \
      --report-file /path/to/AxiomGT1.report.txt \
      --controls-file /path/to/Axiom_ADME.v1.controls.txt \
      --waviness-sd-max 0.5 \
      --mapd-max 1.425 \
      --analysis-name AxiomGT1 \
      --out-dir /path/to/output_directory \
      --log-file /path/to/log_file.txt
      --penn-cnv-output true \
      --penn-cnv-output-plate-corrected true
