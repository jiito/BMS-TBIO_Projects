Introduction

ps-classification reads in the SNP QC metrics tables generated by ps-metrics and performs SNP classification based on customizable criteria, including call rate, FLD, HetSO, and HomRO. It classifies SNPs into seven major categories: "PolyHighResolution","NoMinorHom", "MonoHighResolution", "OffTargetVariant", "CallRateBelowThreshold", "Other", and "UnexpectedGenoFreq". The category "Hemizygous" appears only when hemizygous SNPs are in the metrics file but the special SNPs data is not present in the metrics file. ps-classification also identifies the best probeset per SNP. More details describing the use of ps-classification can be found in the Axiom Genotyping Solution Data Analysis Guide, available at https://tools.thermofisher.com/content/sfs/manuals/axiom_genotyping_solution_analysis_guide.pdf..

Quick Start

This command runs ps-classification on a metrics file named metrics.txt in the directory ./input for a human dataset and writes the classification output to the directory ./classification: 

ps-classification --species-type human --metrics-file ./input/metrics.txt --output-dir ./classification

Details

The input files and options for ps-classification are:

  • metrics-file - The name of the biallelic metrics file generated by ps-metrics
  • multi-metrics-file - The name of the multiallelic metrics file generated by ps-metrics
  • ps2snp-file - The name of a file containing the mapping of probeset IDs to SNP IDs. The column names should be "probeset_id" and "snpid". The user must supply this file when classifying de novo SNPs or any SNPs that have more than one probeset. (Optional)
  • output-dir - Output directory for results files: defaults to current directory.
  • pid-file - The list of probesets that ps-classification should run on. (Optional)

    • The classification thresholds accepted by ps-classification are:

  • species-type - The type of organism genotyped. The current options are "human", non-human "diploid", and "polyploid". Default is "human".
  • output-recommended - Logical value indicating whether to output a list of recommended SNPs to recommended.ps. Default is TRUE.
  • recommended - Flag indicating whether to output a list of converted SNPs to a file recommened.ps: default is FALSE.

    • If a biallelic metrics file is provided, the following classification thresholds are used:

  • cr-cutoff - Threshold for call rate. If not specified, the default for human is 95 and for diploid and polyploid is 97.
  • fld-cutoff - Threshold for FLD: default=3.6.
  • het-so-cutoff - Threshold for HetSO: default=-0.1.
  • het-so-otv-cutoff - Threshold for OTV detection: default=-0.3.
  • hom-ro-1-cutoff - Threshold for HomRO when a SNP has one genotype: default=0.6.
  • hom-ro-2-cutoff - Threshold for HomRO when a SNP has two genotypes: default=0.3.
  • hom-ro-3-cutoff - Threshold for HomRO when a SNP has three genotypes: default=-0.9.
  • hom-ro - Flag indicating whether the metric HomRO is used in classification: default is TRUE.
  • hom-het - Flag indicating whether the metric HomHet is used in classification. The HomHet metric identifies two-cluster SNPs/probesets with one homozygote cluster and one heterozygote cluster. This checks if the minor homozygote cluster is missing, which is unreasonable for highly inbred species (e.g. wheat). This metric should be turned on when classifying SNPs/probesets in highly inbred species: default is TRUE.
  • num-minor-allele-cutoff - Threshold for the number of minor alleles: default=2.
  • priority-order - When performing probeset selection, the best probeset is selected according to the priority order of probeset conversion types. Valid values are sequences of the strings "PolyHighResolution", "NoMinorHom", "OTV", "MonoHighResolution","CallRateBelowThreshold", and "UnexpectedGenoFreq", separated by commas, with each string occuring exactly once: default 'PolyHighResolution,NoMinorHom,OTV,MonoHighResolution,CallRateBelowThreshold'
  • genotype-p-value-cutoff - minimum p-value for a probeset to be recommended instead of UnexpectedGenotypeFrequency: default=0.000001.

    • If the biallelic metrics file contains copy-number aware metrics, the following classification thresholds are used with SNPs with copy-number aware calls:

  • hom-ro-hap-1-cutoff - Threshold for HomRO when a SNP has one haploid genotype: default=0.4.
  • hom-ro-hap-2-cutoff - Threshold for HomRO when a SNP has one haploid genotype: default=0.2.
  • hom-hap-X-cutoff - Maximum distance diploid clusters can be outside of haploid clusters in the X dimension: default=-0.2.
  • hom-hap-Y-lower-cutoff - Maximum distance diploid clusters can be below haploid clusters in the Y dimension: default=1.
  • hom-hap-Y-upper-cutoff - Maximum distance haploid clusters can be above diploid clusters in the Y dimension: default=0.2.
  • CN0-hap-X-cutoff - Maximum distance CN0 clusters can be outside of haploid clusters in the X dimension: default=-0.2.
  • CN0-hap-Y-cutoff - Maximum distance CN0 clusters can be above haploid clusters in the Y dimension: default=0.
  • CN0-dip-X-cutoff - Maximum distance CN0 clusters can be outside of diploid clusters in the X dimension: default=0.2.
  • CN0-dip-Y-cutoff - Maximum distance CN0 clusters can be above diploid clusters in the Y dimension: default=0.

    • If the supplemental filter is turned on, the following classification thresholds are used with SNPs with supplemental metrics:

  • use-supplemental - Flag indicating if the supplemental classification should be performed: default=FALSE.
  • homfld-cut - Minimum HomFLD value for a probeset to be PHR instead of HomHomResolution: default=6.5.
  • n-minor-hom-cut - minimum number of samples in the minor homozygous cluster to be recommended instead of nMinorHom: default=0.
  • hetv-maf-cut - cut-off for het rate versus MAF for UnexpectedHeterozygosity: default=10.
  • variance-class - category types that the supplemental workflow is run on. Takes a single category or a comma separated list: default='PolyHighResolution'
  • bb-varx-cut - minimum X variance value for the BB cluster to be recommended instead of BBvarianceX: default=10.
  • bb-vary-cut - minimum Y variance value for the BB cluster to be recommended instead of BBvarianceY: default=10.
  • ab-varx-cut - minimum X variance value for the AB cluster to be recommended instead of ABvarianceX: default=10.
  • ab-vary-cut - minimum Y variance value for the AB cluster to be recommended instead of ABvarianceY: default=10.
  • aa-varx-cut - minimum X variance value for the AA cluster to be recommended instead of AAvarianceX: default=10.
  • aa-vary-cut - minimum Y variance value for the AA cluster to be recommended instead of AAvarianceY: default=10.
  • bb-varx-z-cut - minimum z-score X variance value for the BB cluster to be recommended instead of BBvarianceX: default=3.
  • bb-vary-z-cut - minimum z-score Y variance value for the BB cluster to be recommended instead of BBvarianceY: default=3.
  • ab-varx-z-cut - minimum z-score X variance value for the AB cluster to be recommended instead of ABvarianceX: default=3.
  • ab-vary-z-cut - minimum z-score Y variance value for the AB cluster to be recommended instead of ABvarianceY: default=3.
  • aa-varx-z-cut - minimum z-score X variance value for the AA cluster to be recommended instead of AAvarianceX: default=3.
  • aa-vary-z-cut - minimum z-score Y variance value for the AA cluster to be recommended instead of AAvarianceY: default=3.
  • bb-varx-z-cut-enabled - flag indicating if the Z thresholds should be on the BB X variances used instead of the variance thresholds with the supplemental workflow: default=TRUE.
  • bb-vary-z-cut-enabled - flag indicating if the Z thresholds should be on the BB Y variances used instead of the variance thresholds with the supplemental workflow: default=TRUE.
  • ab-varx-z-cut-enabled - flag indicating if the Z thresholds should be on the AB X variances used instead of the variance thresholds with the supplemental workflow: default=TRUE.
  • ab-vary-z-cut-enabled - flag indicating if the Z thresholds should be on the AB Y variances used instead of the variance thresholds with the supplemental workflow: default=TRUE.
  • aa-varx-z-cut-enabled - flag indicating if the Z thresholds should be on the AA X variances used instead of the variance thresholds with the supplemental workflow: default=TRUE.
  • aa-vary-z-cut-enabled - flag indicating if the Z thresholds should be on the AA Y variances used instead of the variance thresholds with the supplemental workflow: default=TRUE.

    • If the SSPs file is turned on, ps-classification performs the supplemental workflow and the output the SSPs metrics along with the supplemental metrics:

  • use-ssp - flag indicating if the SSP workflow should be run: default=FALSE.

    • If the biallelic metrics file contains metrics for special SNPs, the following classifiation thresholds are used with non-PAR X, Y, and MT SNPs:

  • hom-ro-hap-1-MTChr-cutoff - minimum HomRO value for MT probesets with 1 haploid cluster: default=0.4.
  • hom-ro-hap-2-MTChr-cutoff - minimum HomRO value for MT probesets with 2 haploid clusters: default=0.2.
  • hom-ro-hap-1-YChr-cutoff - minimum HomRO value for Y probesets with 1 haploid cluster: default=0.4.
  • hom-ro-hap-2-YChr-cutoff - minimum HomRO value for Y probesets with 2 haploid clusters: default=0.2
  • hom-ro-hap-1-XChr-cutoff - minimum HomRO value for non-PAR X probesets with 1 haploid cluster (male samples only): default=0.1.
  • hom-ro-hap-2-XChr-cutoff - minimum HomRO value for non-PAR X probesets with 2 haploid clusters (male samples only): default=0.05.
  • homfld-YChr-cut - minimum HomFLD_hap value for a Y probeset to be PHR instead of Other (male samples only): default=6.5.
  • homfld-XChr-cut - minimum HomFLD_hap value for a non-PAR X probeset to be PHR instead of Other (male samples only): default=6.5.
  • fld-XChr-cut - minimum FLD should be larger than FLD_X.cut for non-PAR X probesets (female samples only): default=4.0.
  • aaf-XChr-cut - maximum difference in A-allele frequency between female and male samples for non-PAR X probesets: default=0.36
  • CN0-hap-X-YChr-cutoff - maximum distance CN0 clusters can be outside of haploid clusters in the X dimension for Y probesets: default=0.1.
  • CN0-hap-Y-YChr-cutoff - maximum distance CN0 clusters can be above haploid clusters in the Y dimension for Y probesets (deprecated): default=0.35.
  • het-so-XChr-cutoff - minimum HetSO should be larger than HetSO_X.cut for non-PAR X probesets (female samples only): default=-0.1.
  • min-YChr-samples-cut - minimum number of male and female samples needed to classify Y probesets using the Euclidean distances thresholds: default=15.

    • If a multiallelic metrics file is provided, the following classification thresholds are used:

  • use-multi-allele - flag indicating if multiallelic classification should be performed: default=FALSE
  • sign-diff-hom-1-cutoff - minimum minSigBgndDiffHom for probesets with 1 cluster: default=0.5.
  • sign-diff-hom-2-cutoff - minimum minSigBgndDiffHom for probesets with 2 clusters: default=0.4.
  • min-mean-cp2-cutoff - minimum MinMean_cp_2 value needed to be PHR: default=9.
  • max-mean-cp2-cutoff - maximum MinMean_cp_2 value needed to be PHR: default=15.
  • genotype-p-value-cutoff - minimum p-value for a probeset to be recommended instead of UnexpectedGenotypeFrequency: default=0.000001.

    • ps-classification produces a performance summary output file (Ps.performance.txt) and several output files, one for each category, with a list of probesets that have been classified in each category. The category probeset files are named after the category (e.g. NoMinorHom.ps). The performance file contains the metrics for each probeset, the probesetâ€™s category (ConversionType), and if the probeset is the best probeset for the SNP. If there are multiple probesets for a SNP, then a ps2snp file must be provided.

  • metrics-file - The name of the file containing output generated by ps-metrics.
  • ps2snp-file (Optional) - The name of a file containing the mapping of probeset IDs to SNP IDs. The column names should be "probeset_id" and "snpid". The user must supply this file when classifying de novo SNPs or any SNPs that have more than one probeset.
  • output-dir - Output directory for results files: defaults to current directory.
  • species-type - The type of organism genotyped. The current options are "human", non-human "diploid", and "polyploid".
  • converted - Flag indicating whether to output a list of converted SNPs to a file converted.ps: default is FALSE.
  • cr-cutoff - Threshold for call rate. If not specified, the default for human is 95 and for diploid and polyploid is 97.

  • hom-ro-1-cutoff - Threshold for HomRO when a SNP has one genotypes: default=0.6.
  • hom-ro-2-cutoff - Threshold for HomRO when a SNP has two genotypes: default=0.3.
  • hom-ro-3-cutoff - Threshold for HomRO when a SNP has three genotypes: default=-0.9.
  • hom-ro - Flag indicating whether the metric HomRO is used in classification: default is TRUE.
  • hom-het - Flag indicating whether the metric HomHet is used in classification. The HomHet metric identifies two-cluster SNPs/probesets with one homozygote cluster and one heterozygote cluster. This checks if the minor homozygote cluster is missing, which is unreasonable for highly inbred species (e.g. wheat). This metric should be turned on when classifying SNPs/probesets in highly inbred species: default is TRUE.
  • num-minor-allele-cutoff - Threshold for the number of minor alleles: default=2.
  • priority-order - When performing probeset selection, the best probeset is selected according to the priority order of probeset conversion types. Valid values are sequences of the strings "PolyHighResolution", "NoMinorHom", "OTV", "MonoHighResolution" and "CallRateBelowThreshold", separated by commas, with each string occuring exactly once: default 'PolyHighResolution,NoMinorHom,OTV,MonoHighResolution,CallRateBelowThreshold'

The metric hom-ro shoud be turned off for organisms where the homozygote clusters may center at zero in the contrast space, particularly polyploid species. If a ps2snp-file is provided, then the best performance probeset will be selected to represent SNPs with multiple probesets. There are seven output files, one for each category, containing lists of probeset IDs belonging to each category. Each file contains a list of probeset IDs belonging to the category. The performance output file summarizes the categorization results across all SNPs.
