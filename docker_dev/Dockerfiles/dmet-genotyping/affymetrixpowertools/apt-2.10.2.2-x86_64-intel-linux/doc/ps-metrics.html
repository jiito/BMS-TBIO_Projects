Introduction

ps-metrics calculates quality control (QC) metrics for SNPs genotyped using Axiom arrays. ps-metrics takes standard genotyping outputs from Axiom arrays and generates a file containing QC metrics for all input SNPs. One metrics file is generated for each type of posteriors file: a biallelic posteriors file results in a biallelic metrics file, and a multiallelic posteriors file results in a multiallelic posteriors file. If the user provides both types of posteriors files, two metrics files are generated. More details describing the use of ps-metrics and detailed explanations for each of the metrics reported can be found in the Axiom Genotyping Solution Data Analysis Guide, available at https://tools.thermofisher.com/content/sfs/manuals/axiom_genotyping_solution_analysis_guide.pdf.

Quick Start

This command runs ps-metrics on the posteriors and calls files in directory ./input and generates the output file metrics.txt: 

ps-metrics --posterior-file ./input/AxiomGT1.snp-posteriors.txt --call-file ./input/AxiomGT1.calls.txt --metrics-file metrics.txt

Details

The following parameters are required to run ps-metrics:

  • posterior-file - The file output from Axiom genotyping which contains posterior summaries of clusters for SNPs (usually named "AxiomGT1.snp-posteriors.txt" or "AxiomGT1.snp-posteriors.multi.txt")..
  • metrics-file - The name of the output metrics file. The user must supply a file name in order to save the output as a separate file, which is read in to other functions such as ps-classification (http://media.affymetrix.com/support/developer/powertools/changelog/Ps_Classification.html).

In addition, one of the following two parameters is required:

  • call-file - The file output by GTC/APT which contains genotype calls for SNPs (usually named "AxiomGT1.calls.txt").
  • chp-files - A text file containing a list of CHP files from which to take calls (one file path per line).

Either the --call-file or the --chp-files parameter must be provided in order to run ps-metrics.
